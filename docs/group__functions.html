◆ generate_permissions_check_query()

String generate_permissions_check_query ( Map  usersColumnsDatatable,
List< List< String >>  selectColumnLabels,
String  tagId 
)

Generate a query to return all permissions (and optional additional columns) for a specific user by Tag ID.

Parameters
usersColumnsDatatableGoogle Visualization query result DataTable
selectColumnLabels(optional) Additional section headers/labels to select
tagIdTag ID of desired user
Returns
String of query in Google Visualization SQL

Steps:

  1. Validate datatable
  2. Search for label that starts with "Permissions", count how many columns precede it (non-permissions columns) and follow it (permissions columns only).
  3. Add provided selectedColumnLabels labels to the select portion of the query
  4. Add permissions column labels to the select portion of the query
  5. Find column ID for Tag ID column and add it to the select portion of the query
  6. Execute permissions check query

Firebase Function HTTPS handler.

Parameters
requestHTTP request
responseHTTP response

Steps:

  • (Check for external ping to keep function "warm", exit 200 immediately)
  • (Check for CORS preflight check, exit 200 immediately)
  1. Extract OAuth access_token from received request headers, use it for Google API requests
  2. Strip /google_apps_script_proxy from received URL path
  3. Create upstream URL path as https://script.google.com/{path}
  4. Send the request, copy upstream response headers to the proxied response
  5. Return the body of the upstream response as the body of the proxied response
  • (In case of failure, return response with exception text. Use the error code from the upstream response if available, otherwise use 500.)

◆ permissions_check_http()

Future<void> permissions_check_http ( GoogleCloudFunctionsRequest  request,
GoogleCloudFunctionsResponse  response 
)

HTTPS handler to query Google Spreadsheet for ACM.User which matches ACM.Activity flatbuffer supplied in request payload.

Parameters
GoogleCloudFunctionsRequestrequest
GoogleCloudFunctionsResponseresponse
Returns
Future which is fulfilled when all child subrequests are complete and ACM.User payload is ready

Steps:

  • (Check for external ping to keep function "warm", exit 200 immediately)
  1. Extract OAuth access_token from received request headers, use it for Google API requests
  2. Parse request body as ACM.Activity flatbuffer
  3. Prepare Firebase Database connection: latestUserRef, activityRef
  4. Check for cached User sheet columns, fetch them if not present
  5. Generate permissions check query in Google Visualization SQL format
  6. Send the query, set "X-DataSource-Auth" header to force JSON format
  7. Check for valid query response
  8. Extract User details from query results
  9. Push matching User details to Firebase
  10. Return matching ACM.User flatbuffer (including permissions info) in response
  11. Append Activity to Google Spreadsheet
  • (In case of failure, return response with exception text. Use the error code from the upstream response if available, otherwise use 500.)
